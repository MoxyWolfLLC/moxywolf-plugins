## Best Practices Across All Formats

### Content Quality
- **Specific over generic** - "Implement MFA via Okta" not "improve security"
- **Data-driven** - Include numbers, percentages, timeframes
- **Action-oriented** - Every item should be executable
- **Context-aware** - Explain why, not just what

### Formatting
- **Scannable** - Headers, bullets, tables, checkboxes
- **Hierarchical** - Clear information architecture
- **Visual breaks** - Use horizontal rules, whitespace
- **Consistent** - Same format for similar elements

### User Experience
- **Progressive disclosure** - Start simple, add detail
- **Multiple entry points** - Table of contents for longer guides
- **Practical examples** - Real scenarios, not hypothetical
- **Clear next steps** - Don't leave users hanging

### Credibility Signals
- **Attribution** - "Based on 22 years experience" or "Analyzed 500 implementations"
- **Recency** - "Updated January 2026" or "Based on 2025 standards"
- **Specificity** - "35,000 weekly users" not "thousands of users"
- **Authority** - Certifications, industry position, client base

## Common Mistakes to Avoid

❌ **Too generic** - "Follow best practices for security"
✅ **Specific** - "Enable MFA with hardware tokens for admin accounts"

❌ **No context** - "Configure firewalls"
✅ **With context** - "Configure firewalls to block ports 1-1024 except 80/443"

❌ **Overwhelming** - 50-page comprehensive guide
✅ **Focused** - One core workflow with 10-20 action items

❌ **No credibility** - Anonymous checklist
✅ **Credible** - "Used by 500+ defense contractors" with attribution

❌ **Static** - One-size-fits-all approach
✅ **Adaptive** - "If Level 1, do X. If Level 2, do Y."
