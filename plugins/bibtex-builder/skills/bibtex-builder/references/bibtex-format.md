# BibTeX Formatting Specification

## Entry Structure

Every entry follows this pattern:

```bibtex
@type{citation_key,
    field1 = {Value One},
    field2 = {Value Two},
    field3 = {Value Three}
}
```

Rules:
- 4-space indentation for all fields
- Field names are lowercase
- Values wrapped in curly braces `{}`
- Comma after every field value EXCEPT the last field
- No trailing comma on the last field
- Opening brace on same line as @type
- Closing brace on its own line, no indentation

## Field Ordering

Fields appear in this order (omit fields that don't apply):

1. `author`
2. `title`
3. `journal` / `booktitle` / `howpublished`
4. `year`
5. `volume`
6. `number`
7. `pages`
8. `publisher`
9. `doi`
10. `url`
11. `urldate` (access date, format: YYYY-MM-DD)
12. `abstract`
13. `keywords`
14. `note`

The `abstract` field is always second-to-last or last.

## Special Character Escaping

BibTeX requires escaping these characters inside field values:

| Character | Escape |
|-----------|--------|
| `&` | `\&` |
| `%` | `\%` |
| `$` | `\$` |
| `#` | `\#` |
| `_` | `\_` |
| `{` | `\{` (when not used for grouping) |
| `}` | `\}` (when not used for grouping) |
| `~` | `\~{}` |
| `^` | `\^{}` |

## Author Formatting

- Multiple authors separated by ` and `: `{Smith, John and Doe, Jane}`
- Format: `{LastName, FirstName}` for each author
- For organizations: `{{National Institute of Standards and Technology}}`
  (double braces prevent BibTeX from treating "and" as a separator)

## Title Formatting

- Wrap words that must keep capitalization in braces: `{A Study of {NIST} Framework {CMMC} Requirements}`
- Acronyms, proper nouns, and technical terms get inner braces

## Entry Types Reference

| Type | Use For |
|------|---------|
| `@article` | Journal or magazine articles |
| `@inproceedings` | Conference papers |
| `@techreport` | Technical reports, whitepapers, standards documents |
| `@misc` | Blog posts, web pages, general online content |
| `@book` | Complete books |
| `@incollection` | Book chapters |
| `@phdthesis` | Doctoral dissertations |
| `@mastersthesis` | Master's theses |
| `@manual` | Technical manuals, documentation |

## File Header

Start every generated .bib file with a comment block:

```bibtex
% Bibliography generated by BibTeX Builder
% Date: YYYY-MM-DD
% Entries: N
% Source: [description of input — URLs, enriched file, etc.]

```

## Complete Example

```bibtex
% Bibliography generated by BibTeX Builder
% Date: 2026-04-08
% Entries: 2
% Source: URL list (2 URLs)

@article{smith_resilience_2024,
    author = {Smith, John and Doe, Jane},
    title = {Building Cyber Resilience in {Critical Infrastructure}: A Framework Approach},
    journal = {Journal of Cybersecurity},
    year = {2024},
    volume = {12},
    number = {3},
    pages = {45--67},
    doi = {10.1234/jcs.2024.0045},
    url = {https://example.com/cyber-resilience},
    urldate = {2026-04-08},
    abstract = {Organizations managing critical infrastructure face escalating cyber threats that demand resilience beyond traditional perimeter defenses. This study proposes a layered framework integrating continuous monitoring, automated incident response, and supply-chain verification to reduce mean time to recovery by 60\%. The findings suggest that resilience-oriented architectures outperform reactive security models in both cost efficiency and operational continuity.}
}

@misc{nist_zero_trust_2023,
    author = {{National Institute of Standards and Technology}},
    title = {Zero Trust Architecture: Implementation Guidance for Federal Agencies},
    howpublished = {NIST Special Publication},
    year = {2023},
    url = {https://example.com/nist-zta},
    urldate = {2026-04-08},
    abstract = {Federal agencies transitioning to zero trust must address identity verification, micro-segmentation, and continuous authorization across hybrid environments. This guidance provides a phased migration path from traditional network perimeters to identity-centric security, with specific milestones tied to OMB M-22-09 compliance deadlines. Practical implementation patterns for agencies at different maturity levels are included alongside risk-assessment templates.}
}
```
